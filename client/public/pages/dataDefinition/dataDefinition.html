In order to feed the system with mining-data it is necessary to describe their basic properties.
The Display-Name Property will be used to extract from an item a string-representation. Therefore it should be unique, but must not.
The Key Properties are used to identify an item.

Example:
If you have items like {display: "Butter", name: "butter"} you would need to have Display Name Property as "display" and one Key Property as "name".
Note, you can define multiple Key Properties by just adding them.
Item Definition

Property used to present an item.

A word of care. Changes in mining-data definition will require to remove all former mining-data and model instances from database. Nevertheless, after changing these settings, you can always upload historic data from Mining Data's upload facilities.
